Безопасность SafeKeys

Трехуровневая защита

1. Уникальный ключ (SK-...)
160 бит энтропии • Для входа с любого устройства

2. Пароль доступа
SHA-256 хеш на сервере • Защита от перебора

3. Мастер-пароль
AES-256 шифрование • Zero-Knowledge

Криптография

• AES-256-GCM — шифрование данных

• PBKDF2 — 100,000 итераций

• SHA-256 — хеш пароля

Что хранится на сервере

Хранится: Уникальный ключ (SK-...)

Хранится: SHA-256 хеш пароля

Хранится: Зашифрованные данные

Не хранится: Пароль расшифровки

Не хранится: Расшифрованные данные

Важно

Сохраните ваш ключ, пароль доступа и мастер-пароль.

Без них доступ к данным невозможен.

Восстановление невозможно (Zero-Knowledge).

Настройка синхронизации

1

Выберите или создайте файл .key внутри папки Google Drive, Dropbox или Яндекс.Диска

2

SafeKeys будет автоматически обновлять этот файл при каждом изменении

3

Облако само синхронизирует файл между всеми вашими устройствами

Привязка к устройству

Зашифровать базу дополнительным ключом, который хранится только в этом браузере. Если вашу базу украдут, её невозможно будет открыть на другом устройстве даже с паролем.

Режим инкогнито

Скрывать пароли, 2FA коды и данные карт в интерфейсе. Данные можно будет только скопировать, но не увидеть глазами.

История паролей

Автоматически сохранять старые версии паролей при их изменении. (Работает только для новых изменений)

Защита от перебора

Если кто-то получил ваш ключ и пытается подобрать пароль, SafeKeys ограничит попытки входа.

Локальная защита активна. После настройки сервера попытки будут считаться и там.
